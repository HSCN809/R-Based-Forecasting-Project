1 Project Overview

This project develops a reproducible R-based forecasting workflow using a TÜİK time series. The selected variable is the monthly rate of change in the Agricultural Products Producer Price Index. The goal is to compare alternative forecasting methods and forecast the next monthly period after the latest available TÜİK observation.

8.1 Naïve Forecasting

The naïve method uses the most recent actual value as the one-step-ahead forecast.

method_lookup <- setNames(forecast_results$methods, vapply(forecast_results$methods, `[[`, character(1), "method"))
naive_result <- method_lookup[["Naïve Forecasting"]]
save_forecast_plot(test_data, naive_result$forecast, naive_result$method, "outputs/figures/naive_forecast_plot.png")

10 Interpretation of Results

The selected TÜİK variable represents the monthly percentage change in the Agricultural Products Producer Price Index. This variable is suitable for forecasting because it is numeric, regularly observed at monthly frequency, and available over a long ordered time span from 2010-02 to 2026-04.

The exploratory analysis shows volatile monthly changes with recurring month-level differences. Seasonal Indices performs better than the competing methods in the holdout comparison, while trend-only and decomposition-based alternatives produce larger aggregate errors or stronger tracking-signal imbalance. The final forecast of -0.6638 percent for 2026-05 means that the selected method expects a small negative monthly change in the index for the next period after the latest available TÜİK observation.

11 Limitations

The series measures monthly percentage change, which can be volatile and negative. This makes multiplicative decomposition unsuitable. The model comparison is based on a 24-month holdout window, so the results may change after later TÜİK revisions or unusual economic shocks. The notebook uses the live TÜİK dataflow verified through tuikr::statistical_tables(), but the SDMX download helper tuikr::statistical_data() returned HTTP 401 in the local environment, so the verified dataflow is accessed through the TÜİK Data Browser JSON endpoint in reproducible R code. This fallback still satisfies the course-approved requirement that data access must happen inside R without manual download or manually prepared data files.
